# BMad Help - Guide d'utilisation

Bienvenue dans **BMad Method** ! Ce workflow vous aide à naviguer dans le framework de développement agile piloté par l'IA.

## 🎯 Commandes principales

### `/bmad-help` - Aide générale
Utilisez cette commande pour obtenir de l'aide sur ce qu'il faut faire ensuite dans votre projet.

## 📋 Workflows disponibles

### Phase 1 : Analyse
- **`/brainstorming`** - Session de brainstorming avec techniques créatives
- **`/create-brief`** - Créer un brief de projet
- **`/create-prd`** - Créer un Product Requirements Document
- **`/create-ux-design`** - Créer des designs UX

### Phase 2 : Planification
- **`/create-architecture`** - Créer l'architecture du projet
- **`/create-epics-and-stories`** - Créer des epics et user stories
- **`/check-implementation-readiness`** - Vérifier si le projet est prêt pour l'implémentation

### Phase 3 : Implémentation rapide (Quick Flow)
- **`/quick-dev`** - Développement rapide avec validation intégrée
- **`/quick-spec`** - Création rapide de spécifications techniques

### Phase 4 : Implémentation structurée
- **`/sprint-planning`** - Planification de sprint
- **`/create-story`** - Créer une user story
- **`/dev-story`** - Développer une story
- **`/code-review`** - Revue de code
- **`/retrospective`** - Rétrospective de sprint

### Documentation
- **`/document-project`** - Documenter le projet complet
- **`/generate-project-context`** - Générer le contexte du projet

## 👥 Agents disponibles

- **@pm** - Product Manager - Gestion de produit et priorisation
- **@analyst** - Business Analyst - Analyse des besoins
- **@architect** - Architecte - Design d'architecture
- **@dev** - Développeur - Implémentation de code
- **@qa** - QA Engineer - Tests et qualité
- **@ux-designer** - UX Designer - Design d'expérience utilisateur
- **@sm** - Scrum Master - Facilitation agile
- **@tech-writer** - Technical Writer - Documentation
- **@quick-flow-solo-dev** - Développeur solo pour Quick Flow

## 🎉 Party Mode

Utilisez **`/party-mode`** pour inviter plusieurs agents à collaborer sur un sujet.

## 🚀 Démarrage recommandé

Pour votre projet **Génération XML**, voici les étapes recommandées :

### 1. Si vous commencez un nouveau projet
1. **`/brainstorming`** - Brainstormer sur les améliorations possibles
2. **`/create-architecture`** - Documenter l'architecture actuelle
3. **`/generate-project-context`** - Générer le contexte du projet

### 2. Si vous développez des fonctionnalités
1. **`/quick-dev`** - Pour des développements rapides
2. **`/create-story`** - Pour créer des user stories
3. **`/dev-story`** - Pour implémenter des stories

### 3. Si vous documentez le projet
1. **`/document-project`** - Documentation complète
2. **`/generate-project-context`** - Contexte du projet

## 💡 Conseils

1. **Commencez petit** - Utilisez `/quick-dev` pour des changements rapides
2. **Utilisez les agents** - Mentionnez @agent pour expertise spécialisée
3. **Suivez le flux** - Les workflows vous guident étape par étape
4. **Demandez de l'aide** - `/bmad-help [votre question]` à tout moment

## 🎯 Prochaines étapes pour votre projet

Votre projet **Génération XML** dispose déjà d'une bonne architecture. Voici ce que vous pourriez faire :

1. **Documenter** : `/generate-project-context` pour créer une vue d'ensemble
2. **Améliorer** : `/brainstorming` pour identifier des améliorations
3. **Tester** : `/qa-generate-e2e-tests` pour ajouter des tests
4. **Développer** : `/quick-dev` pour ajouter de nouvelles fonctionnalités
